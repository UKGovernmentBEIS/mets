Accessibility statement for Manage Your UK Emissions Trading Scheme reporting service

This accessibility statement applies to the Manage your UK Emissions Trading Scheme reporting service website.

This website is run by the Department for Energy Security and Net Zero (DESNZ). We want as many people as possible to be able to use this website. For example, that means you should be able to:

  • change colours, contrast levels and fonts
  • zoom in up to 300% without the text spilling off the screen
  • navigate most of the website using just a keyboard
  • navigate most of the website using speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

We have also made the website text as simple as possible to understand.

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this website is

We know some parts of this website are not fully accessible:

  • there is a short timeout on the page
  • some user interface elements do not have appropriate or sufficiently informative labels
  • navigation via headings may be impeded
  • users of speech recognition software may be unable to interact with all page elements
  • you cannot skip to the main content when using an Android screen reader
  • some user interface elements do not have a visible focus indicator

Feedback and contact information

Contact us by emailing Mets@energysecurity.gov.uk

The website contains an interactive business process rather than static information, so the contents are not available in alternative formats.

Reporting accessibility problems with this website

We are always looking to improve the accessibility of this website. If you find any problems not listed on this page or think we’re not meeting accessibility requirements, contact: Mets@energysecurity.gov.uk

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’). If you are not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about this website’s accessibility

The Department for Energy Security and Net Zero is committed to making its website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

This website is partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard, due to the non-compliances and exemptions listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

Some buttons lack an associated label or similar, this may result in screen readers not providing context for the button. This fails WCAG 2.1 success criterion 1.1.1 (Non-Text Content) We plan to resolve this by October 2023.

Semantic markup is not used appropriately to structure tables, headings and lists. Empty elements are present which may be announced in an unexpected way. This fails WCAG 2.1 success criterion 1.3.1 (Info and Relationships). We plan to resolve this by October 2023.

Related form elements are not grouped which causes some checkboxes to be announced without context. This fails WCAG 2.1 success criterion 1.3.1 (Info and Relationships) We plan to resolve this by October 2023.

An autocomplete attribute is incorrectly present on many fields which may cause confusion when completing forms. This fails WCAG 2.1 success criterion 1.3.5 (Info and Relationships)

We plan to resolve this by October 2023.

A short time limit is present on the site with a very limited way to extend it. This fails WCAG 2.1 success criterion 2.2.1 (Timing Adjustable We plan to resolve this by October 2023.

A correctly formed heading structure is not present to facilitate navigation across the site, with empty heading levels present. This fails WCAG 2.1 success criterion 2.4.1 (Bypass Blocks). We plan to resolve this by October 2023.

Page titles are not always appropriate or informative. The Accounts page has a title which does not reflect the page content. This fails WCAG 2.1 success criterion 2.4.2 (Page Titled). We plan to resolve this by October 2023.

The focus order is not logical when using Android screen readers to navigate the page. This fails WCAG 2.1 success criterion 2.4.3 (Focus Order) We plan to resolve this by October 2023.

Many identical links are not distinguished by accessible labels. ‘Change’ and ‘Delete’ links are present with no additional information to distinguish them. This fails WCAG 2.1 success criterion 2.4.4 Link Purpose (In Context). We plan to resolve this by October 2023.

Some pages cannot be found through more than one type of navigation. This fails WCAG 2.1 success criterion 2.4.5 (Multiple Ways). We plan to resolve this by October 2023.

Some headings and labels are not sufficiently informative. This fails WCAG 2.1 success criterion 2.4.6 (Headings and Labels). We plan to resolve this by October 2023.

A visible focus indicator is not present on the upload file functionality. This fails WCAG 2.1 success criterion 2.4.7 (Focus Visible). We plan to resolve this by October 2023.

The accessible label on table filter elements do not match the beginning of the displayed name which may cause navigation issues for users of speech recognition software. This fails WCAG 2.1 success criterion 2.5.3 Link Purpose (In Context). We plan to resolve this by October 2023.

Multiple validation errors are present across multiple webpages. This may result in some content announcing incorrectly. This fails WCAG 2.1 success criterion 4.1.1 (Parsing) We plan to resolve this by October 2023.

The ‘Skip to Main Content’ Call-To-Action does not function when using an Android screen reader. This fails WCAG 2.1 success criterion 4.1.2 (Name, Role, Value) We plan to resolve this by October 2023.

Markup has not been used to aid accessibility, with elements not using allowed ARIA attributes and ARIA roles not conforming to valid values. Elements assigned invalid ARIA role values are not interpreted by assistive technologies as intended. This fails WCAG 2.1 success criterion 4.1.2 (Name, Role, Value). We plan to resolve this by October 2023.

Changes in state may not always be announced by screen readers. This includes form submissions. This fails WCAG 2.1 success criterion 4.1.3 (Status Messages) We plan to resolve this by October 2023.

Disproportionate burden

Interactive tools

The upload file function is unavailable whilst navigating using speech recognition software. This is a known flaw of speech recognition software.

We’ve assessed the cost of fixing the issues with interactive tools and transactions. We believe that doing so now would be a disproportionate burden within the meaning of the accessibility regulations.
